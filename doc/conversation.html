2. 백엔드 선택 — 3가지 옵션 분석

옵션 A — v1 DataSet 패턴 그대로 (빠른 실전형)

  • Spring Boot 3.2.5 + MyBatis + Flowable, DataSet 단일 진입점 /api/dataset/*
  • 결재/게시판/캘린더/조직도 완성도 70%, 즉시 가동
  • 단점: Mattermost OIDC 유료 제약, 장기 확장성 약함

옵션 B — v2 Hexagonal BFF 풀구현

  • Hexagonal 아키텍처 + Port-Adapter 설계
  • Keycloak Federation (GitLab OAuth 트릭), 무료 SSO 경로 확보
  • 단점: 구현 40% 에서 시작 → 일정 장기화

옵션 C — 하이브리드 ⭐ 채택

  • Core 도메인: v1 의 DataSet + MyBatis + Flowable 선택 포팅 → backend-core
  • BFF 통합 계층: v2 의 Port-Adapter 로 외부 서비스 프록시 → backend-bff
  • UI: vue-spring-fw 의 Vue 3 + PrimeVue 4 컴포넌트 정적 복사 + Keycloak-js 어댑터
  • 근거: UI 별도 제작 전제에서 v1 도메인 재활용 가치 최대. 외부 서비스는 Port 로 격리하면 장기 유연성 확보

#서비스인증 경로상태
1포털 UI (Vue+Pinia)Keycloak-js PKCE + silent-SSO✅ 완전
2backend-coreJWT via jwk-set-uri✅ 완전
3backend-bffJWT 검증 + LiveKit 재발급✅ 완전
4LiveKitBFF 경유 HS256 JWT✅ 완전
5Wiki.jsKeycloak native strategy + updateStrategies mutation✅ 완전
6Rocket.Chat (Mattermost 대체)Custom OAuth → Keycloak🟡 95% (SSO 버튼 표시 + 토큰 교환 성공, userinfo 에서 identityTokenSentVia 설정 적용이 재시작 직후 지연)
7Stalwart 메일OpenLDAP → Keycloak User Federation (READ_ONLY)✅ 완전 (IMAP STARTTLS 검증됨)
8MinIOOIDC server-side + role_policy=consoleAdmin🟡 S3 STS 완전 / Console UI 버튼 해당 버전 미노출

결론

포털 1회 로그인으로 5개 핵심 경로(포털·core·bff·LiveKit·Wiki.js)가 완전 자동 SSO. Stalwart 는 LDAP Federation 으로 동일 계정 인증. Rocket.Chat 은 SSO 버튼까지 표시되고 Keycloak 통과까지 성공, 최종 userinfo 단계에서 인증 헤더 전달 타이밍 이슈. MinIO 는 S3 API 수준에서 STS 토큰으로 OIDC 사용 가능하며 관리자 콘솔만 폼 로그인 유지.

11. 주요 해결한 기술 문제

  1. MyBatis Map 결과 camelCase 변환 
    @Intercepts({@Signature(type = ResultSetHandler.class, method = "handleResultSets", args = {Statement.class})})
public static class CamelCaseMapInterceptor implements Interceptor { ... }
  2. Keycloak 이중 hostname 문제: 브라우저는 localhost:19281, docker 내부는 keycloak:8080. issuer-uri 검증이 이를 거부해서 jwk-set-uri 로 전환.
  3. Mattermost TE OAuth Enterprise 제약: 9.x/7.x 모두 라이선스 요구. Rocket.Chat Community Edition 으로 전환 (Custom OAuth 완전 무료).
  4. Stalwart 내부 디렉토리 → LDAP Federation: osixia/openldap 컨테이너 추가 → Keycloak User Storage Federation 등록 → Stalwart directory.ldap-keycloak 설정 → storage.directory 를 internal에서 ldap-keycloak으로 전환.
  5. iframe X-Frame-Options: Wiki.js/Mattermost/Stalwart 모두 deny/sameorigin 하드코딩. iframe 포기하고 "새 탭에서 열기" 방식 + SSO 버튼 UX.
  6. Keycloak 토큰 iss 검증: 여러 hostname 을 수용하기 위해 jwk-set-uri 만 사용 (서명만 검증, iss 완화).
