1. Rocket.Chat userinfo 401 완전 해결

디버깅 여정

  1. Custom OAuth 설정 Mongo 직접 주입 (Accounts_OAuth_Custom-Keycloak-*)
  2. Service name 케이스 불일치 (keycloak vs Keycloak) 해결
  3. identityTokenSentVia=header 반영 검증 (디버그 패치로 확인)
  4. 토큰 scope 에 openid 포함 확인 (requestPermissions 추가)
  5. 최종 원인 특정: 토큰 iss 클레임 불일치
핵심 원인:
브라우저가 http://localhost:19281 에서 authorize → 토큰의 iss = http://localhost:19281/realms/openplatform-v3
Rocket.Chat 서버가 http://keycloak:8080(Docker 내부) 에서 userinfo 호출 → Keycloak 이 현재 hostname(keycloak:8080) 과 토큰 iss(localhost:19281) 불일치로 invalid_token 응답.

해결 방법

KC_HOSTNAME_URL: "http://localhost:19281"
KC_HOSTNAME_ADMIN_URL: "http://localhost:19281"

고정 frontend URL 로 Keycloak 이 항상 iss=localhost:19281 로 토큰 발급하도록 강제 → 내/외부 hostname 불일치 해소.

추가 조치

  • Email 2FA 비활성화 (Rocket.Chat 이 SSO 프로비저닝 사용자에게 2FA 요구하던 문제)
  • Playwright 로 "Keycloak SSO" 버튼 클릭 → Rocket.Chat 홈("Rocket.Chat 방문을 환영합니다!") 진입 확인
  • Mongo users 컬렉션 확인: user1@v3.local 에 services.keycloak 필드 확인 (SSO 프로비저닝 성공)
